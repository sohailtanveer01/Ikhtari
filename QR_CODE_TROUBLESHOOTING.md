# QR Code Troubleshooting Guide

## Issue: "No usable data found" when scanning QR code

### Root Cause

Your app uses **`expo-dev-client`**, which requires a **custom development build**. The QR codes generated by Expo work differently for dev client apps than they do for Expo Go.

---

## ✅ Solution 1: Use Development Build (Recommended)

Since your app uses `expo-dev-client`, you need to:

### Step 1: Build Development Version

```bash
# Build development version for your device
eas build --profile development --platform ios

# Or for Android
eas build --profile development --platform android
```

### Step 2: Install Development Build

1. Download the build from the EAS build URL
2. Install it on your device (iOS: TestFlight or direct install, Android: APK)
3. Open the development build app on your device

### Step 3: Connect to Dev Server

1. Start the Expo dev server:
   ```bash
   npx expo start --dev-client
   ```

2. In the terminal, you'll see options:
   - **Press `s`** to switch to development build connection mode
   - Scan the QR code with your **device's camera app** (not Expo Go)
   - Or manually enter the connection URL shown in the terminal

### Step 4: Connect Manually (Alternative)

If QR code doesn't work, you can connect manually:

1. Make sure your device and computer are on the **same WiFi network**
2. In the Expo dev server terminal, look for:
   ```
   Metro waiting on exp://192.168.x.x:8081
   ```
3. Open the development build app on your device
4. Shake the device or use a gesture to open the dev menu
5. Tap **"Enter URL manually"**
6. Enter: `exp://192.168.x.x:8081` (use the IP shown in terminal)

---

## ✅ Solution 2: Use Tunnel Mode

If you're having network connectivity issues:

```bash
# Start with tunnel mode (works even on different networks)
npx expo start --dev-client --tunnel
```

This creates a tunnel that works across different networks, but it's slower.

---

## ✅ Solution 3: Check Network Configuration

Make sure your device and computer are on the same network:

1. **Check WiFi:**
   - Device and computer must be on the same WiFi network
   - Some corporate/public WiFi blocks device-to-device communication

2. **Check Firewall:**
   - Ensure your computer's firewall isn't blocking port 8081
   - On Mac: System Settings → Network → Firewall → Options

3. **Check IP Address:**
   - Terminal shows: `Metro waiting on exp://192.168.x.x:8081`
   - Make sure this IP matches your computer's local IP
   - You can find your IP with: `ipconfig getifaddr en0` (Mac) or `ipconfig` (Windows)

---

## ❌ Why Expo Go Won't Work

**Expo Go cannot scan QR codes for dev client apps** because:
- Your app uses native modules that Expo Go doesn't support:
  - `@react-native-google-signin/google-signin`
  - `expo-apple-authentication`
  - `react-native-maps`
  - Custom native code

- Dev client apps need to be built with your specific native dependencies

---

## 🔍 Quick Diagnosis

### Check if you're using Expo Go:
- ❌ **Expo Go app** = Won't work with your app
- ✅ **Development build app** = Will work

### Check Expo server output:
```bash
npx expo start --dev-client
```

You should see:
```
› Metro waiting on exp://192.168.x.x:8081
› Scan the QR code above with Expo Go (Android) or the Camera app (iOS)
```

If you see "Expo Go" mentioned, press `s` to switch to dev client mode.

## 🚀 Recommended Workflow

### For Development:

1. **Build once:**
   ```bash
   eas build --profile development --platform ios
   ```

2. **Install on device** (TestFlight or direct install)

3. **Start dev server:**
   ```bash
   npx expo start --dev-client
   ```

4. **Connect:**
   - Scan QR code with device camera
   - Or manually enter URL in dev menu
   - Or use tunnel mode: `--tunnel`

### For Quick Testing:

1. **Use iOS Simulator:**
   ```bash
   npx expo run:ios
   ```

2. **Use Android Emulator:**
   ```bash
   npx expo run:android
   ```

---

## 📱 Alternative: Use Simulator/Emulator

If QR code scanning continues to be problematic, use simulators:

```bash
# iOS Simulator (Mac only)
npx expo run:ios

# Android Emulator
npx expo run:android
```

This builds and runs the app directly in the simulator/emulator.

---

## 🛠️ Common Fixes

### 1. Restart Expo Server
```bash
# Kill the current server (Ctrl+C)
# Then restart:
npx expo start --dev-client --clear
```

### 2. Clear Cache
```bash
npx expo start --dev-client --clear
```

### 3. Check Connection URL
In the terminal, verify the connection URL is correct and accessible from your device.

### 4. Rebuild Development Build
If the development build is outdated:
```bash
eas build --profile development --platform ios --clear-cache
```

---

## 📝 Notes

- **Development builds** work like production apps but with dev tools
- **QR codes** are just URLs - you can manually enter them too
- **Tunnel mode** is slower but works across networks
- **Simulators/Emulators** don't need QR codes - they connect automatically
